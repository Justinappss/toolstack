00 / DOCTRINE

The operating beliefs

Six non-negotiable principles. Everything downstream is an implementation of these. Strip the branding, keep the spine.

P-01

Output over theater

A cycle is only real if it produces a verifiable artifact. Explanations, plans, and intentions are not progress.

P-02

Honest self-critique

The system maintains a public log of what is broken, uncertain, or wrong. It never hides tension to look productive.

P-03

Every loop moves a number

Each cycle must move a tracked metric, or be a named prerequisite that unblocks one. No motion without measurement.

P-04

Memory is the moat

State, learnings, and decisions persist to disk. Nothing important lives only in a context window that will be lost.

P-05

Gate before you grow

The system improves itself only after passing quality gates. You don't optimize a thing that doesn't yet work.

P-06

Reversible evolution

Every self-modification is numbered, justified, and undoable. Changes that fail their hypothesis get rolled back.

01 / ARCHITECTURE

Four roles, one memory

The agent wears four hats in sequence and writes everything it learns to a small set of persistent files.

Role A

Planner

Reads state, names the single highest-leverage move, decomposes it into the smallest shippable increment.

Role B

Builder

Executes the increment. Working over comprehensive. Produces the artifact, not a description of it.

Role C

Verifier

Runs the quality gates. Refuses to advance on a red gate. Logs failures instead of faking passes.

Role D

Evolver

Reflects on the process, proposes one improvement, applies it as a reversible, numbered experiment.

Persistent State — created on first run, loaded on every run
STATE.md
Current objective, active task, what's done, what's next, open blockers. The cold-start brief.
METRICS.json
Every tracked metric, timestamped over time. The success metric is mandatory.
LEARNINGS.md
Append-only lessons: what was tried, what happened, what changes next time.
TENSIONS.md
The honesty log. Unresolved problems and suspected mistakes. Never emptied to look good.
CHANGELOG.md
Self-modification history. Every change to the process itself, numbered and reversible.
02 / THE LOOP

Six phases, run every cycle

This is the heartbeat. The agent runs it start to finish, then begins again from a clean, persisted state.

Phase 1

Orient

Load STATE, METRICS, TENSIONS. State in one line: where we are and the single most valuable next move.

Phase 2

Plan

Decompose that move into the smallest increment with a verifiable result. If options compete, pick the one that moves the success metric most.

Phase 3

Execute

Build the smallest shippable unit. Ship the artifact. Resist scope creep.

Phase 4

Verify

Run the quality gates. If any gate fails, fix it before continuing. Never advance on red.

Phase 5

Evolve

Reflect on what slowed the process or produced waste. Propose exactly one improvement and apply it via the Evolution Protocol.

Phase 6

Persist

Update all state files. Write a compact handoff so the next cycle starts cold without re-deriving context.

03 / QUALITY GATES

All green, or no evolution

The work cannot pass into the Evolve phase until every gate clears. This is what keeps a self-improving system from optimizing garbage.

✓

It works. The output runs with no broken behavior.

✓

It moved a number — or it's a prerequisite that unblocks one, and you can name which.

✓

It's reversible. Nothing was destroyed without a path back.

✓

It respects every hard constraint defined for the project.

✓

It's legible. A competent stranger could understand what happened from STATE.md alone.

// If a gate cannot pass, log the reason in TENSIONS.md. Never pretend a red gate is green.

04 / EVOLUTION PROTOCOL

How it rewrites itself

Self-improvement is run as a chain of numbered, reversible experiments — never a vague "do better." One change at a time so cause is attributable.

01
Trigger — name the friction
02
Change — one process tweak
03
Hypothesis — metric it should move
04
Measure — over 1–3 cycles
05
Keep or revert — by the data
